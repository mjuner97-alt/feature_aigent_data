# 数据领域知识

## 各维度标准格式：
● 版本计划（versionPlan）：标准格式为'xxxx年x月份版本'，例如：2026年6月份版本。当前版本默认：2026年6月份版本。
● 季度（vestQuarter）：标准格式为'xxxx年x季度'，例如：2026年2季度。
● 部门（deptName）：杭州开发一部、杭州开发二部、杭州开发三部、杭州开发四部、杭州开发五部、云计算实验室、杭州技术部、杭州服务支持部
● 小组（groupName）：标准格式'xxxx组'，例如金融市场自营测试组、杭州二部FMBM应用平台组、杭州二部金融市场交易报价组、杭州四部金融市场后台运营开发组、杭州五部普惠大文章线上化组、杭州服务支持部GBC场景创新组、API及高可用保障团队、等以组或团队结尾。
注意：1. 若分析各个小组，该字段为空。
     2. 由于组值不规范，有的可能会包含部门和应用。若调工具时，首先根据部门调工具`deptGroupCheck`，看具体组名。
● 应用（appName）：标准格式'F-xxxx'。例如：F-FMTA、F-BMS-TRADE。约束：1）强制`F-`前缀转换（`FMTA`→`F-FMTA`、`CFBI-DSP`→`F-CFBI-DSP`）。2）禁止数字/中文（‘FMTA1’→ 'F-FMTA1'无效，'F-FMTA'正确）.若分析各个应用，该字段为空。
● 群组（appName）：标准格式 'F-xxxx-xxx'，例如F-ATP-GENPAY。若分析各个群组，该字段为空。
● 产品线（productLine）：标准格式'xxxx产品线'，例如银银产品线、代理国库、个人信贷产品线、法人信贷产品线、缴费产品线、普惠金融、全球市场风险管理应用、金融市场共享数据服务、交行输出、银企产品线、代理财政、代理同业、等。
● 需求项（itemNo）：例如I20260208-0005。若分析各个需求项，该字段为空。
● 人（perName）：xxx，例如xxx。若分析各个人，则该字段为空。
